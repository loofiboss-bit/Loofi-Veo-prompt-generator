# v2.7.0 UI Foundations

## Design Direction

- Visual language: dark cinematic professional, with light theme parity.
- Navigation IA: preserved.
- Primary goals: consistency, hierarchy clarity, predictable overlays, and maintainable theming.

## Foundation Rules

### Spacing

- Base grid: 4px.
- Core containers use `--spacing-4` to `--spacing-8` increments.
- Panel inner spacing defaults to `--spacing-6`.

### Typography

- Default line-height returns to tokenized base (`1.5`) for body text.
- No global forced letter-spacing on all elements.
- Tracking adjustments are opt-in and component-scoped.

### Elevation

- Surface layers:
- Level 0: app background
- Level 1: cards/panels
- Level 2: elevated controls/docks
- Level 3: dialogs and overlays

### Z-index Contract

- `--z-base`: 0
- `--z-fixed`: 1200
- `--z-modal-backdrop`: 1300
- `--z-modal`: 1400
- `--z-overlay`: 1500
- `--z-tutorial`: 1550
- `--z-notification`: 1700

### Surface Roles

- Shell surfaces: sidebar, header, action surfaces.
- Content surfaces: prompt form, summary/output, feature panels.
- Overlay surfaces: dialogs, onboarding overlay, diagnostics/help layers.

## Component Architecture

- New shared `AppDialog` primitive drives backdrop, z-index, focus trap, close behavior, and responsive size.
- Feature dialogs use `AppDialog` with content-only composition.
- Theme source of truth: `ThemeService` + store synchronization.
- Accessibility modes that affect layout are opt-in class toggles.
