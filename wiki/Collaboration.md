# Collaboration

This page describes team-oriented workflow patterns for shared Veo Studio usage.

## Collaboration model

Veo Studio includes collaboration-oriented features and services that support shared editing and review workflows.

## Team operating practices

1. Align on project naming and status conventions.
2. Define role expectations (owner/editor/reviewer).
3. Keep templates and presets synchronized for consistency.
4. Use history and comments for review decisions.

## Session hygiene

- Start with a known project baseline.
- Avoid large unannounced changes in active sessions.
- Group edits by intent for easier conflict resolution.
- Communicate plugin activation/deactivation events.

## Conflict reduction strategy

- Keep edits scoped to specific shots/scenes when possible.
- Commit/serialize major milestones before risky modifications.
- Use history comparison tools to merge and reconcile outcomes.

## Security considerations

- Collaborate only with trusted participants.
- Assume peers can inspect collaboration metadata required for connectivity.
- Do not share sensitive tokens/keys inside shared notes or prompts.
