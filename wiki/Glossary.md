# Glossary

Key terms used across Veo Studio documentation.

| Term            | Meaning                                                                       |
| --------------- | ----------------------------------------------------------------------------- |
| Local-first     | Data and workflow state are primarily stored and processed on the user device |
| Prompt workflow | Structured process to design, generate, and iterate AI-ready prompts          |
| Store           | Zustand-based state container that coordinates UI-facing state/actions        |
| Service         | Singleton business-logic layer handling side effects and integrations         |
| Plugin manifest | Metadata and permission declaration for a plugin package                      |
| Plugin context  | Scoped runtime object exposed to active plugins                               |
| Trust level     | Plugin trust classification (`trusted`, `unsigned`, etc.)                     |
| Safe mode       | Desktop startup fallback triggered after crash-loop detection                 |
| Validation gate | Required quality checks (`lint`, `typecheck`, `test`, `format`)               |
| Feature module  | Domain-oriented folder in `src/features` representing user-facing capability  |
