## 1. Bug Fixes (High Priority)

### 1.1 Memory Leak — videoEditorService.ts:157

**File**: `src/core/services/videoEditorService.ts`
**Problem**: `navigator.serviceWorker.addEventListener('message', ...)` is added inside the `stitchVideos()` Promise (line 157) but never removed. Each call stacks a new listener that persists forever.
**Fix**: Capture the handler as a named function, call `removeEventListener` on both `RENDER_COMPLETE` and `RENDER_ERROR` events.

```typescript
// BEFORE (line 153-169):
return new Promise((resolve, reject) => {
    navigator.serviceWorker.addEventListener('message', (event) => { ... });
});

// AFTER:
return new Promise((resolve, reject) => {
    const handler = (event: MessageEvent) => {
      const { type, payload, progress, status } = event.data;
      if (type === 'RENDER_PROGRESS') {
        if (onProgress) onProgress(`${status} (${Math.round(progress)}%)`);
      } else if (type === 'RENDER_COMPLETE') {
        navigator.serviceWorker.removeEventListener('message', handler);
        resolve(URL.createObjectURL(payload as Blob));
      } else if (type === 'RENDER_ERROR') {
        navigator.serviceWorker.removeEventListener('message', handler);
        reject(new Error(payload));
      }
    };
    navigator.serviceWorker.addEventListener('message', handler);
});
```

### 1.2 Security — process.env Exposure in vite.config.ts

**File**: `vite.config.ts` line 46
**Problem**: `'process.env': process.env` leaks ALL build-time env vars into the client bundle.
**Fix**: Replace with explicit allowlist:

```typescript
// BEFORE:
define: {
    'process.env': process.env,
    'import.meta.env.VITE_APP_VERSION': JSON.stringify(packageJson.version),
},

// AFTER:
define: {
    'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV),
    'import.meta.env.VITE_APP_VERSION': JSON.stringify(packageJson.version),
},
```

Then search the codebase for any `process.env.X` references in src/ that need to become `import.meta.env.VITE_X`.
