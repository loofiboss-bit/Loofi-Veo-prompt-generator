# v1.5.0 Acceptance Checklist

Date created: 2026-02-10
Purpose: Pass/fail criteria for v1.5.0 Performance & Stability implementation tasks.

## Sprint 1 - Foundation

- [ ] Error schema is defined and versioned (`code`, `message`, `stack`, `context`, `correlationId`, `timestamp`).
- [ ] Renderer, async, and IPC errors all reach centralized logging.
- [ ] Error-write IPC is non-blocking and rate-limited/deduplicated.
- [ ] React ErrorBoundary exists for each critical panel and shows recoverable fallback UI.
- [ ] CI has required blocking checks: `npm run lint:ci`, `npm run typecheck`, `npm run test`.
- [ ] Forced throws in each layer are captured without freezing the app.

## Sprint 2 - Performance

- [ ] Zustand `persist.partialize` excludes transient UI state and keeps required domain state.
- [ ] Migration test confirms previous persisted data still hydrates correctly.
- [ ] Profiling service is gated (enabled in dev/flagged mode; off by default in production).
- [ ] `performance.mark/measure` covers startup, prompt generation, timeline scrub, and export start.
- [ ] Batch IPC is used for high-frequency calls with a defined max batch window.
- [ ] Timeline/media cleanup revokes blob URLs only at safe lifecycle points.
- [ ] Baseline-vs-after metrics show improvement (startup, memory, interaction latency).

## Sprint 3 - UX and Stability

- [ ] Safe Mode triggers only after 3 confirmed abnormal exits (not normal close/dev reload).
- [ ] Safe Mode reset flow works after a stable launch.
- [ ] Timeline optimizations (memo + rAF debounce) improve FPS without drag lag.
- [ ] Yjs race conditions are mitigated with AbortController and stale-closure handling.
- [ ] Hotkey conflict resolution respects focused inputs, modals, and reserved platform shortcuts.
- [ ] Stress runs with rapid edits/collaboration join-leave show no crashes/race regressions.

## Sprint 4 - DevOps and Security

- [ ] Bundle-size budget check runs in CI with explicit threshold and clear failure output.
- [ ] PR changelog validation runs with documented exceptions (for docs-only/internal-only changes).
- [ ] Electron hardening baseline is enforced (`contextIsolation: true`, `nodeIntegration: false`, IPC allowlist).
- [ ] `webSecurity` is enabled and verified against current app behavior.
- [ ] Code-signing preparation is documented and staging can produce signed artifacts.
- [ ] Release-candidate pipeline completes end-to-end with all policy checks passing.
